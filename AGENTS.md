# Repository Instructions

## Design Image Assets

When generating or saving design images, concept art, character sheets, UI mockups, or game reference images for this repo, use the repo-local workflow in `.codex/skills/direct-save-design-images/SKILL.md`.

Project design images must be treated as real assets, not chat previews.

- Save final images into `C:\Learning\game-website\assets\design-examples` unless the user names another folder.
- Use clear filenames such as `male-character-turnaround-v1.png`, `female-character-turnaround-v1.png`, `airport-exterior-v1.png`, `restaurant-street-v1.png`, or `phone-ui-map-v1.png`.
- After generating, open the saved file with visual inspection before saying it is done.
- Do not claim an image is saved until the file exists on disk and has been visually verified.
- If an image generator only produces a chat preview without an accessible file, do not substitute a lower-quality recreation or a screenshot without saying so. Use a direct file-writing generation path, or tell the user what is blocking exact save/export.
