# Rishon3D V1.1 — Wave 3: Building Facade Purpose (plan)

Spec: `docs/superpowers/specs/2026-06-18-rishon3d-v1.1-scale-city-identity-design.md`
Branch: `worktree-3d-spike` (do NOT merge/deploy). Work dir: `rishon3d/`.

Goal: buildings are colorful but generic. Give ground floors readable IDENTITIES via
deterministic facade TYPES (restaurant / shop / apartment / office), plus parapets,
doors/entry pads, and apartment balconies. Build on the existing facade system
(`world/facade.ts` + `world/builders.ts` from the World Detail Pass). Keep blocky/voxel,
deterministic, cached textures, flat draw calls. (Airport terminal = a unique authored
landmark, handled in Wave 4 destinations, not a procedural type here.)

P0 (done by orchestrator): `palette.ts` gains `facadeDoor, officeGlass, signWarm,
signCool, signLit, balconyRail, entryPad`.

Two SEQUENTIAL stages (facade.ts then builders.ts — builders consumes facade's new API):

## Stage 1 — `facade.ts` (typed facade textures)
- Add `export type FacadeType = "shop" | "restaurant" | "apartment" | "office"`.
- Extend `facadePattern` / `makeFacadeTexture` to take a `type` (in `FacadeOpts`) and vary:
  - **shop**: bright glass storefront + a colored **sign band** (`signCool`) above the
    storefront + display-window mullions; sparse upper windows.
  - **restaurant**: warm-lit storefront windows + a **sign band** (`signWarm`/`signLit`)
    + a door; warmer glass.
  - **apartment**: regular repeated window grid all floors, smaller windows, modest
    storefront/entry (door, no big glass); this is the "lots of identical windows" look.
  - **office**: uniform **blue glass grid** (`officeGlass`) curtain wall, clean storefront
    lobby (big glass + door), minimal sign.
  - All types: keep the **cornice/parapet** top row; keep a **door** marking in the
    storefront/entry row.
- Keep the existing default behavior + signature back-compatible (default type ~"shop");
  keep the texture cache keyed to include `type`. Keep it pure/node-testable.
- Tests (`test/facade.test.ts`): per-type determinism; sign band present for shop/
  restaurant; office glass dominant + uniform; apartment has more window rows / no big
  storefront glass; door present; cache differentiates types.

## Stage 2 — `builders.ts` (assign types + type geometry)
- `facadeTypeFor(def): FacadeType` — deterministic from id hash + size heuristic: tall
  (height >~16) -> office/apartment; mid -> apartment/office; short/wide -> shop/restaurant.
- Pass the type into `makeFacadeTexture`. Add type-specific GEOMETRY (merged where
  possible, still one mesh/building + small extras):
  - **parapet / roof cap**: a thin raised rim box on the roof for all non-house buildings
    (kills the blank-top look).
  - **door + entry pad**: a dark `facadeDoor` panel at the street-facing base + a small
    `entryPad` slab on the ground in front (oriented to the nearest road if cheap, else
    +Z front).
  - **balconies (apartment only)**: a few small `balconyRail` boxes protruding on the
    front face at upper floors.
  - bias `awningStyle` so **shop/restaurant** types more often get awnings (they already
    exist via `makeAwnings`); keep awnings off office/apartment mostly.
- Keep house path unchanged. Keep `makeBuilding` returning a single Object3D the World
  adds (group is fine if multiple pieces). Keep perf: extras merged/instanced; do not
  explode draw calls (a per-building group of a few merged meshes is acceptable, or merge
  parapet/door into the building where feasible).
- Tests (`test/builders.test.ts`): `facadeTypeFor` determinism + heuristic buckets;
  building output includes a parapet + door; apartment buildings include balcony geo;
  houses unchanged.
